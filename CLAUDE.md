## Build System

**Generate project files (required before first build or after premake5.lua changes):**
```batch
cd Engine\Build
generate_project.bat
```
This runs `premake5 vs2022` and produces Visual Studio 2022 solution/project files.

**Build:** Open `Engine/Engine.sln` in Visual Studio 2022. The Launcher project is the startup project.

**Configurations:**
- `Game Debug` / `Game Release` — runs standalone game
- `Editor Debug` / `Editor Release` / `Editor ASan Debug` — runs with in-engine editor
- `PROFILE Game/Editor Debug/Release` — profiling builds (Tracy)

Output: `Bin/` directory. Intermediate files: `Temp/`.

**Run tests:** Build and run the `Unit Test` project (Google Test). Test files are in `Engine/Unit Test/`.

**PhysX setup** (only needed once if submodule not compiled):
```batch
Engine\Build\internal_scripts\generate_physx_files.bat
Engine\Build\internal_scripts\compile_physx.bat
```

**Required environment:**
- Visual Studio 2022
- Vulkan SDK (`VULKAN_SDK` environment variable must be set)
- assimp (via vcpkg — see `vcpkg_info.txt`)
- PhysX (git submodule — `git submodule update --init --recursive`)

## Architecture Overview

The codebase is a C++23 Vulkan rendering engine with an editor. It is split into several Visual Studio projects: **Engine** (static lib), **Editor** (static lib), **Game** (static lib), **Launcher** (entry point), **ImGui**, **Spirv-Reflect**, **ReflectionGenerator**, and **Unit Test**.

### Entry Point & Startup

`Launcher/Main.cpp` initializes the `Engine`, `Game`, and optionally `Editor` (when `EDITOR` is defined). A tick loop calls `engine.Tick()` which drives time, input, world logic, and rendering each frame.

### System Architecture

Engine subsystems inherit from `System` and are managed by `SystemManager`. Access them globally via `Engine::GetEngineSystem<T>()`. Key systems:
- **Core:** `Time`, `Input`, `ThreadPool`, `Filewatcher`, `Console`
- **Rendering:** `RenderSystem`, `MeshSystem`, `TextureSystem`, `GPUResourceManager`, `StagingSystem`
- **Vulkan backend:** `VulkanContext` (device/queue init), `VulkanSwapChain`, `VulkanAllocator` (VMA)
- **Physics:** `PhysicsSystem` (PhysX wrapper)
- **Scene:** `GPUSceneSystem`, `PointLightSystem`

### World & Actor/Component System

- `World` — container for all actors; drives tick and physics
- `Actor` — base game object, owns `Component`s
- `Component` — behavior unit attached to actors; built-ins include `Transform`, `Camera`, `RenderComponent`, `Rigidbody`, collider types, lights, `CharacterController`

### Rendering Pipeline

Rendering is driven by `RenderSystem::Tick()` which executes a `RenderGraph`. The render graph handles automatic Vulkan barrier insertion and resource state tracking. Render passes implement `IRenderPass` and are categorized as `GraphicsPass`, `ComputePass`, or `TransitionPass`.

Active passes (in order):
1. `IndirectCullPass` — GPU compute culling
2. `MainPass` — PBR geometry
3. `SkyboxPass`
4. `DebugPass`
5. `ImGuiPass`
6. `CopyToSwapchainPass`

Shaders are HLSL, compiled to SPIR-V. Reflection data is extracted via `spirv-reflect`. Shader cache files live in `Bin/Engine/Shaders/*.hlsl.cache`.

### Asset System

`AssetRegistry` handles loading and caching. Asset types: `Model`, `Texture`, `TextureCube`, `Shader`, `Material`, `JsonAsset`, `World`. Assets use a custom binary serialization format built on the reflection system.

### Reflection System

Runtime type information is code-generated by `ReflectionGenerator` as a prebuild step, producing `Launcher/GeneratedReflectionData.hpp`. Fields marked with the `META()` macro are included. The `ReflectionSystem` provides `Type` objects with `Field` and `Method` accessors. Use `Engine::GetReflectionSystem()` to look up types at runtime.

### Memory & Containers

Custom smart pointers (`UniquePtr`, `SharedPtr`, `WeakPtr`) and containers (`List<T>`, `SegmentedList<T>`, `MutexList<T>`) are used throughout. mimalloc handles heap allocation. VMA (`VulkanAllocator`) manages GPU memory.

### GPU Debugging

NVIDIA Aftermath is integrated for GPU crash diagnostics. Crash data is logged on device-lost events.

### Profiling

Tracy profiler is included. Use `PROFILE Game/Editor` configurations to enable it.
